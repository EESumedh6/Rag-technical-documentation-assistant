Pydantic Beginner to Intermediate Guide

Introduction

Pydantic is a Python library used for data validation and parsing.
It uses Python type hints to validate data automatically.

Pydantic is widely used with:
- FastAPI
- APIs
- AI applications
- Backend systems
- Configuration management

Installation

pip install pydantic

BaseModel

The core component in Pydantic is BaseModel.

from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int

Creating Objects

user = User(name="Alice", age=25)

Automatic Type Conversion

user = User(name="Bob", age="30")

Pydantic converts string to integer automatically.

Validation Errors

Invalid values raise validation errors.

User(name="John", age="hello")

Field Types

Pydantic supports many field types.

- str
- int
- float
- bool
- list
- dict
- datetime

Example

class Product(BaseModel):

Optional Fields

from typing import Optional

class User(BaseModel):
    email: Optional[str] = None

Default Values

class Settings(BaseModel):
    debug: bool = False

Lists

from typing import List

class Blog(BaseModel):

Dictionaries

from typing import Dict

class Config(BaseModel):

Nested Models

class Address(BaseModel):
    city: str
    country: str

class Customer(BaseModel):
    name: str
    address: Address

Nested Model Example

customer = Customer(
    name="Alice",
    address={
        "city": "Mumbai",
        "country": "India"
    }
)

Serialization

Convert model to dictionary.

user.model_dump()

Convert model to JSON.

user.model_dump_json()

Parsing JSON

json_data = '{"name":"Alice","age":22}'

User.model_validate_json(json_data)

Field Validation

from pydantic import Field

class Product(BaseModel):
    price: float = Field(gt=0)

Field Constraints

gt means greater than.
lt means less than.

String Constraints

class User(BaseModel):
    username: str = Field(min_length=3)

Number Constraints

class Data(BaseModel):
    age: int = Field(gt=18)

Custom Validators

from pydantic import field_validator

class User(BaseModel):
    age: int

    @field_validator("age")
    def validate_age(cls, value):
        if value < 18:
            raise ValueError("Must be adult")
        return value

Email Validation

from pydantic import EmailStr

class User(BaseModel):
    email: EmailStr

pip install email-validator

Aliases

from pydantic import Field

Enum Support

from enum import Enum

class Role(str, Enum):

class Account(BaseModel):

Model Inheritance

class BaseUser(BaseModel):
    name: str

class AdminUser(BaseUser):
    permissions: list[str]

ORM Mode

class User(BaseModel):
    id: int
    name: str

    model_config = {
        "from_attributes": True
    }

Validation From Objects

class UserObject:
    def __init__(self):
        self.id = 1
        self.name = "Alice"

User.model_validate(UserObject())

Strict Types

from pydantic import StrictInt

class Data(BaseModel):
    age: StrictInt

Strict types prevent automatic conversion.

Computed Fields

class Rectangle(BaseModel):
    width: int
    height: int

    @property
    def area(self):
        return self.width * self.height

FastAPI Integration

FastAPI heavily depends on Pydantic.

from fastapi import FastAPI

app = FastAPI()

class User(BaseModel):
    name: str
    age: int

@app.post("/users")
def create_user(user: User):
    return user
